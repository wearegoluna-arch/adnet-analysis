Section 02

Overlap & Deduplication Risk

RichAds + ROIAds
🔴 HIGH RISK

Identical formats (push + pop), highly likely overlapping device push databases, same iGaming vertical. High probability of paying for the same user twice. Never run on the same GEOs without S2S deduplication.

AADS + Blockchain-Ads
🟠 MEDIUM-HIGH

Both target Web3/crypto publisher inventory. Slise and Blockchain-Ads may share wallet/explorer site inventory. Must confirm publisher list overlap before running concurrently.

RichAds + AADS
🟡 LOW-MEDIUM

Some gaming site overlap but different inventory pools — RichAds mainstream web, AADS crypto-native. Safer to run concurrently with standard frequency capping.

ROIAds + AADS
🟢 LOW

Different formats and inventory sources entirely. Safest pair to run simultaneously. No special deduplication setup required beyond standard S2S tracking.

⚠ Concurrency Rule
Never run RichAds + ROIAds on the same GEO without frequency capping and S2S deduplication at pixel level. Flag AADS + Blockchain-Ads to disclose publisher list overlap before allocating budget to both simultaneously.
